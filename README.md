## ğŸ— Architecture & Principles I follow

### Frontend architecture
- **Feature-based structure** (organized by domains, not file types)
- **Separation of concerns**: UI, state, and data access are not a soup
- **State strategy**: local state first, then shared (Zustand), then server cache
- **Reusable UI**: components are products, not disposable code

### Code quality
- **Readability > cleverness**
- **Small functions & clear naming**
- **Error boundaries + predictable failure modes**
- **Testing pyramid**: unit tests for logic, integration for flows, e2e for critical paths

### Performance
- Avoid unnecessary re-renders (memoization with purpose)
- Good defaults: code splitting, lazy routes, image optimization
- Measure first: Lighthouse/Web Vitals, profiling, bundle analysis
